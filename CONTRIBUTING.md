# 贡献指南

感谢关注并准备为 vnpy-grid 做贡献。为确保协作高效，请遵循以下流程：

## 分支与提交

- 在独立分支开发，分支命名：`feature/xxx`、`fix/yyy`、`docs/zzz`。
- 提交信息遵循约定式提交：`feat: …`、`fix: …`、`docs: …`、`test: …`、`refactor: …`。
- 保持单次提交原子化，便于审阅与回滚。

## 代码规范

- Python 版本：3.11+。
- 使用 ruff/black 统一风格与静态检查。
- 采用卫语句处理错误，禁止吞掉异常。
- 优先向量化与批处理，避免低效循环。

## 测试

- 使用 pytest，遵循 AAA（Arrange-Act-Assert）。
- 新增/修改核心逻辑需附带测试，覆盖率目标 ≥ 80%。
- 禁止使用虚构或模拟数据；示例与测试需可用真实数据源或明确数据准备步骤。

## 提交 PR

- 填写完整的 PR 模板，描述动机、实现与风险。
- 关联相关 Issue，并提供重现实例或脚本。
- 通过本地 `pytest -q` 与 ruff/black 检查。

## 治理与行为

- 遵守 `CODE_OF_CONDUCT.md`。
- 尊重他人时间与精力，最小化破坏性改动。
- 优先讨论架构与接口，再推进实现。
